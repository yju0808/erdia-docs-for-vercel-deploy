# 📏 사용 한도

Erdia를 도입할 때 알아두면 좋은 한도와 제약을 정리했어요. 현재는 Free 플랜 기준으로 동작하며, Pro 플랜이 출시되면 함께 업데이트될 예정이에요.

![사용량 배지 예시](/images/example.png)

| 구분 | Free | 비고 |
| --- | --- | --- |
| 프로젝트 수 | 무제한 | 대시보드에서 모두 관리 가능 |
| AI 채팅 | 50회/일 | 자정 이후 자동 초기화 |
| 스키마 내보내기 | 무제한 | 옵션 선택 후 즉시 다운로드 |
| 스키마 가져오기 | 무제한 | MySQL · PostgreSQL · Prisma · JPA 지원 |
| PNG 내보내기 | 준비 중 | 버튼은 보이지만 아직 동작하지 않아요 |

<Callout type="info">
AI 채팅 한도는 계정 단위로 관리돼요. 여러 프로젝트에서 작업하더라도 총합 50회까지 사용할 수 있으니, 중요한 요청을 우선순위에 따라 정리해 주세요.
</Callout>

## 🤖 AI 채팅 한도 관리 팁

![AI 한도 경고 예시](/images/example.png)

1. 긴 설명을 여러 번 보내기보다, 한 번에 명확한 요구사항을 전달해요.
2. 기존 다이어그램을 수정하고 싶다면 `현재 ERD 상태:`로 시작해 변경하고 싶은 부분만 설명해 주세요.
3. 한도가 다 소진됐다면 JSON으로 내보낸 뒤 직접 다듬어 `스키마 가져오기`로 반영할 수 있어요.

<Callout type="warning">
한도를 초과하면 추가 메시지가 실패하거나, `Failed to fetch response from server.`와 같은 안내가 나타날 수 있어요. 하루가 지나면 자동으로 복구됩니다.
</Callout>

## 🗂 프로젝트와 자동 저장

![자동 저장 배지](/images/example.png)

- 프로젝트는 개수 제한 없이 만들 수 있어요. 이름만 겹치지 않으면 원하는 만큼 관리할 수 있죠.
- 편집기에서는 변경 후 약 1초 이내에 자동 저장이 실행되고, 상단 툴바에서 `저장 중...` → `저장 완료`로 상태가 바뀝니다.
- 대시보드 카드에는 최신 저장 시간이 표시되니, 협업 시 마지막 편집 시점을 쉽게 확인할 수 있어요.

<Callout type="success">
자동 저장이 실패하는 경우는 드물지만, 네트워크가 불안정할 때는 잠시 대기한 뒤 `대시보드` 버튼으로 이동해 저장 여부를 확인해 주세요.
</Callout>

## 🔄 가져오기 · 내보내기 제약

![가져오기 옵션](/images/example.png)

- SQL, Prisma, JPA 파일은 크기 제한 없이 업로드할 수 있어요. 다만, 문법 오류가 있으면 파싱이 실패하고 토스트로 안내돼요.
- 가져올 때 `엔티티 자동 배치`와 `관계 가져오기` 옵션을 꺼두면 구조를 부분적으로만 반영할 수 있어요.
- 내보낼 때는 외래키, 인덱스, 타임스탬프 포함 여부를 각각 선택할 수 있어 팀 규칙에 맞춰 조정하기 쉬워요.

<Callout type="info">
PNG 내보내기는 버튼만 먼저 공개된 상태예요. 실제 이미지 다운로드는 업데이트 후 사용할 수 있으니, 그전까지는 브라우저 캡처 기능을 이용해 주세요.
</Callout>
