# 🚀 Erdia 빠른 시작 가이드

이 문서는 처음 Erdia를 사용하는 분이 **첫 프로젝트를 생성하고 편집기 기본 흐름을 익히는 데** 필요한 모든 단계를 제공합니다. 모든 설명은 실제 화면 순서를 기반으로 작성되었습니다.

## ✨ 회원가입 없이 바로 시작

Erdia는 현재 계정 시스템을 제공하지 않습니다. 홈 화면의 **`시작하기` 버튼**을 누르면 즉시 프로젝트 대시보드가 열리고, 모든 프로젝트 데이터는 브라우저의 로컬 스토리지에 안전하게 보관됩니다.

<Callout type="warning">
같은 브라우저와 기기에서만 프로젝트가 유지됩니다. 새 기기에서 이어서 작업하려면 JSON이나 SQL로 내보낸 뒤 가져오기를 이용하세요.
</Callout>

## 🧭 대시보드 살펴보기

![프로젝트 대시보드 목업](/images/example.png)

대시보드는 최근 수정일 기준으로 프로젝트를 정렬하고 검색할 수 있는 공간입니다.

1. **상단 네비게이션** – 좌측 로고를 누르면 랜딩 페이지로, 우측 버튼을 누르면 다시 랜딩으로 돌아갈 수 있습니다.
2. **검색 입력창** – 프로젝트 이름 또는 설명을 실시간으로 필터링합니다.
3. **`새 프로젝트` 버튼** – 이름과 설명을 입력하는 다이얼로그를 엽니다.
4. **프로젝트 카드** – 엔티티/관계 수, 최근 수정일, 설명을 한눈에 보여줍니다. 카드 우측의 점 3개 버튼을 통해 수정·삭제 메뉴에 접근합니다.

## 🆕 새 프로젝트 만들기

1. 대시보드 우측 상단의 **`새 프로젝트` 버튼**을 클릭합니다.
2. 프로젝트 이름과 설명을 입력합니다. 이름은 필수이며, 기존 프로젝트와 중복되면 에러 토스트가 표시됩니다.
3. **`생성`**을 누르면 프로젝트가 저장되고 편집기로 자동 이동합니다.
4. 이동과 동시에 상단에 성공 토스트가 나타나 프로젝트 생성이 완료되었음을 알려줍니다.

<Callout type="info">
모든 프로젝트는 `project-타임스탬프` 형식의 고유 ID로 관리됩니다. 생성 직후 `프로젝트 상세` 화면으로 이동해 즉시 작업을 시작할 수 있습니다.
</Callout>

## 🖥️ 편집기 기본 구조

편집기는 화면을 가득 채우는 캔버스와 세 가지 보조 영역으로 구성됩니다.

- **상단 툴바** – 프로젝트 이름·설명을 표시하고, `엔티티 추가`, `자동 정렬`, `더보기` 메뉴(내보내기·가져오기·전체 삭제 등)를 제공합니다.
- **우측 하단 액션 버튼** – 채팅 기록 열기, 테마 토글(라이트/다크), 빠른 엔티티 추가 버튼이 배치되어 있습니다.
- **하단 중앙 채팅 입력창** – 자연어로 AI 어시스턴트에 요청을 보낼 수 있습니다. 응답이 수신되면 자동으로 채팅 사이드바가 열립니다.
- **캔버스 본문** – 엔티티를 드래그로 이동하고, 우클릭 컨텍스트 메뉴로 엔티티 추가·삭제, 관계 편집 등을 수행합니다.

## 💡 첫 엔티티 추가하기

1. 상단 툴바의 **`엔티티 추가`** 버튼을 누르거나, 캔버스 빈 공간에서 우클릭 후 **`여기에 엔티티 추가`**를 선택하세요.
2. 새 엔티티는 `New Entity`라는 이름과 `id` 기본키 속성을 포함한 상태로 생성됩니다.
3. 엔티티 제목을 더블클릭하면 이름을 바로 수정할 수 있고, 각 속성 행의 연필 아이콘을 눌러 타입, 기본값, 기본키/외래키 여부를 변경할 수 있습니다.
4. 엔티티 사이를 드래그해 관계를 만들면 관계 타입(1:1, 1:N 등)과 식별 관계 여부를 선택하는 다이얼로그가 나타납니다.

## 💾 저장 상태 확인

- 편집기 상단 툴바 오른쪽에 있는 표시등이 **`저장 중...` → `저장 완료`** 상태로 바뀌며 자동 저장 여부를 안내합니다.
- 저장은 노드나 관계가 변경된 뒤 약간의 지연을 두고 실행되며, 브라우저를 새로고침해도 최근 상태가 유지됩니다.

<Callout type="success">
저장 표시가 사라지지 않을 때는 잠시 기다렸다가 페이지를 새로고침하세요. 네트워크 연결이 없어도 로컬 스토리지에 저장되므로 변경 사항이 사라지지 않습니다.
</Callout>
