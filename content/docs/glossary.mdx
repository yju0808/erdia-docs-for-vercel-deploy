# 용어 설명

ERD와 데이터베이스 설계에 사용되는 주요 용어들을 알아보세요.

## ERD 기본 용어

### ERD (Entity-Relationship Diagram)

엔티티 관계 다이어그램. 데이터베이스의 구조를 시각적으로 표현한 다이어그램입니다.

**용도:**
- 데이터베이스 설계 및 문서화
- 팀원 간 데이터 구조 공유
- 요구사항 분석 및 검증

---

### 엔티티 (Entity)

데이터베이스의 **테이블**을 의미합니다.

**예시:**
- `User` - 사용자 정보를 저장하는 테이블
- `Product` - 상품 정보를 저장하는 테이블
- `Order` - 주문 정보를 저장하는 테이블

<Callout type="info">
**명명 규칙**: 엔티티 이름은 단수형(User)으로 작성하며, PascalCase를 사용합니다.
</Callout>

---

### 속성 (Attribute)

테이블의 **컬럼(필드)**을 의미합니다.

**예시:**
- `id` - 고유 식별자
- `email` - 이메일 주소
- `name` - 이름

**속성의 구성 요소:**
- 속성 이름 (name)
- 데이터 타입 (type)
- 제약조건 (constraints)

---

### 관계 (Relationship)

엔티티 간의 **연결**을 의미합니다.

**관계의 종류:**
- 1:1 (One-to-One)
- 1:N (One-to-Many)
- N:M (Many-to-Many)

---

## 관계 유형

### 1:1 관계 (One-to-One)

한 엔티티가 다른 엔티티 **하나**와만 연결되는 관계입니다.

| 엔티티 A | 관계 | 엔티티 B | 설명 |
|----------|------|----------|------|
| User | ↔ | Profile | 한 사용자는 하나의 프로필만 가짐 |
| Person | ↔ | Passport | 한 사람은 하나의 여권만 가짐 |

**구현 방법:**
```sql
CREATE TABLE User (
  id INT PRIMARY KEY
);

CREATE TABLE Profile (
  id INT PRIMARY KEY,
  user_id INT UNIQUE,  -- UNIQUE 제약조건으로 1:1 보장
  FOREIGN KEY (user_id) REFERENCES User(id)
);
```

---

### 1:N 관계 (One-to-Many)

한 엔티티가 다른 엔티티 **여러 개**와 연결되는 관계입니다.

| 엔티티 A | 관계 | 엔티티 B | 설명 |
|----------|------|----------|------|
| User | → | Post | 한 사용자가 여러 게시글 작성 |
| Category | → | Product | 한 카테고리에 여러 상품 |
| Order | → | OrderItem | 한 주문에 여러 주문 상품 |

**구현 방법:**
```sql
CREATE TABLE User (
  id INT PRIMARY KEY
);

CREATE TABLE Post (
  id INT PRIMARY KEY,
  user_id INT,  -- Foreign Key
  FOREIGN KEY (user_id) REFERENCES User(id)
);
```

<Callout type="success">
**가장 일반적인 관계**: 1:N 관계는 데이터베이스에서 가장 흔하게 사용됩니다.
</Callout>

---

### N:M 관계 (Many-to-Many)

여러 엔티티가 여러 엔티티와 연결되는 관계입니다.

| 엔티티 A | 관계 | 엔티티 B | 설명 |
|----------|------|----------|------|
| Post | ↔ | Tag | 게시글은 여러 태그, 태그는 여러 게시글 |
| Student | ↔ | Course | 학생은 여러 과목, 과목은 여러 학생 |
| Product | ↔ | Cart | 상품은 여러 장바구니, 장바구니는 여러 상품 |

**구현 방법: 중간 테이블 사용**
```sql
CREATE TABLE Post (
  id INT PRIMARY KEY
);

CREATE TABLE Tag (
  id INT PRIMARY KEY
);

-- 중간 테이블 (Junction Table)
CREATE TABLE PostTag (
  post_id INT,
  tag_id INT,
  PRIMARY KEY (post_id, tag_id),
  FOREIGN KEY (post_id) REFERENCES Post(id),
  FOREIGN KEY (tag_id) REFERENCES Tag(id)
);
```

## 제약조건 (Constraints)

### Primary Key (PK)

테이블의 각 행을 **고유하게 식별**하는 컬럼입니다.

**특징:**
- ✅ NULL 값을 가질 수 없음 (NOT NULL)
- ✅ 중복 값을 가질 수 없음 (UNIQUE)
- ✅ 테이블당 하나만 존재

**예시:**
```sql
CREATE TABLE User (
  id INT PRIMARY KEY,  -- Primary Key
  email VARCHAR(255)
);
```

---

### Foreign Key (FK)

다른 테이블의 **Primary Key를 참조**하는 컬럼입니다.

**특징:**
- ✅ 참조 무결성 보장
- ✅ 부모 테이블의 값만 허용
- ✅ 관계 설정에 사용

**예시:**
```sql
CREATE TABLE Post (
  id INT PRIMARY KEY,
  user_id INT,  -- Foreign Key
  FOREIGN KEY (user_id) REFERENCES User(id)
);
```

<Callout type="warning">
**참조 무결성**: Foreign Key는 부모 테이블에 존재하는 값만 가질 수 있습니다.
</Callout>

---

### UNIQUE

컬럼 값이 **중복되지 않음**을 보장합니다.

**예시:**
```sql
CREATE TABLE User (
  id INT PRIMARY KEY,
  email VARCHAR(255) UNIQUE  -- 이메일 중복 방지
);
```

---

### NOT NULL

컬럼 값이 **NULL이 될 수 없음**을 보장합니다.

**예시:**
```sql
CREATE TABLE User (
  id INT PRIMARY KEY,
  email VARCHAR(255) NOT NULL  -- 이메일 필수
);
```

---

### DEFAULT

컬럼의 **기본값**을 설정합니다.

**예시:**
```sql
CREATE TABLE Post (
  id INT PRIMARY KEY,
  status VARCHAR(20) DEFAULT 'draft',  -- 기본값: draft
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 데이터 타입

### 숫자 타입

| 타입 | MySQL | PostgreSQL | 범위 | 용도 |
|------|-------|------------|------|------|
| **정수** | INT | INTEGER | -2,147,483,648 ~ 2,147,483,647 | 일반 정수 |
| **큰 정수** | BIGINT | BIGINT | -9,223,372,036,854,775,808 ~ ... | 대용량 ID |
| **자동 증가** | AUTO_INCREMENT | SERIAL | 1, 2, 3... | Primary Key |
| **소수** | DECIMAL(10,2) | NUMERIC(10,2) | 정밀한 소수 | 금액, 가격 |
| **실수** | FLOAT | REAL | 근사치 소수 | 과학 계산 |

**예시:**
```sql
CREATE TABLE Product (
  id INT AUTO_INCREMENT PRIMARY KEY,  -- 자동 증가 ID
  price DECIMAL(10,2),  -- 가격 (소수점 2자리)
  stock INT  -- 재고 수량
);
```

---

### 문자열 타입

| 타입 | MySQL | PostgreSQL | 최대 길이 | 용도 |
|------|-------|------------|----------|------|
| **짧은 문자열** | VARCHAR(n) | VARCHAR(n) | n자 | 이름, 이메일 |
| **긴 문자열** | TEXT | TEXT | 65,535자 | 본문, 설명 |
| **고정 길이** | CHAR(n) | CHAR(n) | n자 (고정) | 코드, 상태 |

**예시:**
```sql
CREATE TABLE Post (
  id INT PRIMARY KEY,
  title VARCHAR(255),  -- 제목 (최대 255자)
  content TEXT  -- 본문 (긴 텍스트)
);
```

<Callout type="info">
**VARCHAR vs TEXT**: 짧은 문자열(255자 이하)은 VARCHAR, 긴 문자열은 TEXT를 사용하세요.
</Callout>

---

### 날짜/시간 타입

| 타입 | MySQL | PostgreSQL | 형식 | 용도 |
|------|-------|------------|------|------|
| **날짜** | DATE | DATE | YYYY-MM-DD | 생년월일 |
| **시간** | TIME | TIME | HH:MM:SS | 근무 시간 |
| **날짜+시간** | DATETIME | TIMESTAMP | YYYY-MM-DD HH:MM:SS | 생성/수정 시간 |
| **타임스탬프** | TIMESTAMP | TIMESTAMP | 자동 갱신 가능 | created_at |

**예시:**
```sql
CREATE TABLE User (
  id INT PRIMARY KEY,
  birth_date DATE,  -- 생년월일
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,  -- 생성 시간
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP  -- 수정 시간
);
```

---

### 기타 타입

| 타입 | MySQL | PostgreSQL | 설명 | 예시 |
|------|-------|------------|------|------|
| **불린** | BOOLEAN / TINYINT(1) | BOOLEAN | 참/거짓 | is_active |
| **ENUM** | ENUM('a','b','c') | - | 정해진 값 중 선택 | status |
| **JSON** | JSON | JSONB | JSON 데이터 | metadata |

**예시:**
```sql
CREATE TABLE Order (
  id INT PRIMARY KEY,
  status ENUM('pending', 'paid', 'shipped', 'delivered'),  -- 주문 상태
  is_active BOOLEAN DEFAULT TRUE,  -- 활성 여부
  metadata JSON  -- 추가 정보
);
```

## 관계 설정 용어

### 식별 관계 (Identifying Relationship)

부모 테이블의 Primary Key가 자식 테이블의 **Primary Key에 포함**되는 관계입니다.

**예시:**
```sql
CREATE TABLE Order (
  id INT PRIMARY KEY
);

CREATE TABLE OrderItem (
  order_id INT,
  item_number INT,
  PRIMARY KEY (order_id, item_number),  -- order_id가 PK의 일부
  FOREIGN KEY (order_id) REFERENCES Order(id)
);
```

**특징:**
- 강한 종속성
- 부모 없이 자식 존재 불가

---

### 비식별 관계 (Non-Identifying Relationship)

부모 테이블의 Primary Key가 자식 테이블의 **일반 Foreign Key**인 관계입니다.

**예시:**
```sql
CREATE TABLE User (
  id INT PRIMARY KEY
);

CREATE TABLE Post (
  id INT PRIMARY KEY,  -- 독립적인 PK
  user_id INT,  -- 일반 FK
  FOREIGN KEY (user_id) REFERENCES User(id)
);
```

**특징:**
- 약한 종속성
- 부모와 독립적인 식별자

<Callout type="success">
**권장**: 대부분의 경우 **비식별 관계**를 사용하는 것이 유연합니다.
</Callout>

## 데이터베이스 용어

### DDL (Data Definition Language)

데이터베이스 **구조**를 정의하는 언어입니다.

**명령어:**
- `CREATE TABLE` - 테이블 생성
- `ALTER TABLE` - 테이블 수정
- `DROP TABLE` - 테이블 삭제

---

### DML (Data Manipulation Language)

데이터를 **조작**하는 언어입니다.

**명령어:**
- `INSERT` - 데이터 삽입
- `UPDATE` - 데이터 수정
- `DELETE` - 데이터 삭제
- `SELECT` - 데이터 조회

---

### 스키마 (Schema)

데이터베이스 내의 **네임스페이스**입니다.

**예시:**
- MySQL: `mydb.User` (mydb 스키마의 User 테이블)
- PostgreSQL: `public.User` (public 스키마의 User 테이블)

---

### 인덱스 (Index)

데이터 **검색 속도**를 향상시키는 자료구조입니다.

**종류:**
- Primary Key Index (자동 생성)
- Unique Index (중복 방지)
- 일반 Index (검색 최적화)
