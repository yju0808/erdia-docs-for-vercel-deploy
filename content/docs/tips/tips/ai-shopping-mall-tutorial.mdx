# AI로 쇼핑몰 ERD 5분 만에 만들기

"쇼핑몰 데이터베이스 설계하는데 며칠이 걸렸어요..." 😰

## 🎯 목표

이번 튜토리얼에서는 다음 기능을 가진 쇼핑몰 ERD를 만들어봅니다:

- ✅ 회원 가입 및 로그인
- ✅ 상품 카탈로그

## 1단계: 핵심 엔티티 생성 (1분)

먼저 AI에게 기본 구조를 요청합니다.

```
💬 "쇼핑몰 ERD 만들어줘.
필요한 테이블:
- User: 회원 정보
- Product: 상품 정보
- Category: 상품 카테고리
- Cart: 장바구니
- Order: 주문
- OrderItem: 주문 상품 상세"
```

**AI가 자동으로 생성:**
- 6개 테이블과 기본 컬럼
- Primary Key 설정
- 기본 관계 연결

<Callout type="success">
**결과**: 핵심 테이블 6개가 캔버스에 생성됩니다!
</Callout>

## 2단계: 상세 컬럼 추가 (2분)

이제 각 테이블에 필요한 컬럼을 추가합니다.

### User 테이블

```
💬 "User 테이블에 다음 컬럼 추가:
- phone: VARCHAR(20)
- address: TEXT
- postal_code: VARCHAR(10)
- email은 unique로 설정"
```

### Product 테이블

```
💬 "Product 테이블에:
- category_id: INT (Category 테이블 참조)
- price: DECIMAL(10,2)
- stock: INT
- description: TEXT
- image_url: VARCHAR(255)
- is_active: BOOLEAN"
```

### Order 테이블

```
💬 "Order 테이블에:
- user_id: INT (User 참조)
- total_amount: DECIMAL(10,2)
- status: ENUM('pending', 'paid', 'shipped', 'delivered', 'cancelled')
- shipping_address: TEXT
- payment_method: VARCHAR(50)"
```

```
- user_id: INT
```

```
User와 Product의 중간 테이블로 만들어줘"
```

## 4단계: 관계 최적화 (1분)

### 카테고리 계층 구조

```
💬 "Category에 parent_id 추가해서 계층 구조 만들어줘.
예: 전자제품 > 노트북 > 게이밍 노트북"
```

**AI가 생성:**
```sql
CREATE TABLE Category (
  id INT PRIMARY KEY,
  name VARCHAR(100),
  parent_id INT,
  FOREIGN KEY (parent_id) REFERENCES Category(id)
);
```

## 완성된 ERD 구조

```
User (회원)
├── Cart (장바구니)
├── Order (주문)
│   └── OrderItem (주문상품)
├── Review (리뷰)

Product (상품)
├── Category (카테고리)
├── Cart
├── OrderItem
└── Review
```

**총 테이블 수**: 7개  
**총 관계 수**: 10개  
**소요 시간**: 5분
