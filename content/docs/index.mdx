# 👋 Erdia에 오신 걸 환영해요

Erdia는 "아이디어를 구조화하고 팀에 전달한다"라는 개발 업무 흐름을 그대로 옮겨 놓은 ERD 설계 툴이에요. 자연어와 시각 편집을 함께 쓰면서 **로그인 → 프로젝트 준비 → 캔버스 편집 → 산출물 공유**까지 한 번에 처리할 수 있습니다.

![랜딩 화면에서 살펴보기](/images/example.png)

---

## 🚀 처음 들어왔을 때 해야 할 일

1. 홈 화면 우측의 **무료로 시작하기**를 누르면 Google OAuth로 연결돼요. 인증이 끝나면 자동으로 대시보드로 이동합니다.
2. 대시보드 상단의 검색창과 `새 프로젝트` 버튼으로 작업 범위를 정리해요. 이름은 중복 검사를 거치므로 실수로 덮어쓸 걱정이 없어요.
3. 프로젝트를 만들면 즉시 편집기로 전환돼요. 상단 툴바에서 저장 상태(`저장 중...` → `저장 완료`)를 확인하면서 마음 편히 작업을 이어가세요.

<Callout type="warning">
로그인 화면의 **게스트 모드로 둘러보기** 버튼은 아직 준비 중이에요. 실사용은 Google 로그인으로 진행해 주세요.
</Callout>

---

## 🎨 캔버스에서 구조를 만드는 흐름

![대시보드 검색과 카드 예시](/images/example.png)

- **엔티티 추가** 버튼 또는 우클릭 메뉴로 테이블 카드를 배치해요. 기본으로 `id` 컬럼이 생성되고, 더블클릭하면 이름을 바로 바꿀 수 있습니다.
- 각 컬럼 옆 연필 아이콘으로 타입을 고치거나 기본키·외래키·NULL 여부를 토글할 수 있어요. 삭제 시에는 역할에 따라 경고 문구가 먼저 뜹니다.
- 컬럼 핸들을 드래그하면 관계 생성 마법사가 열려요. 카디널리티(1:1, 1:N 등)와 식별 여부를 고르고, 연결할 컬럼을 지정하면 즉시 선이 그려집니다.
- 자동 정렬을 누르면 겹치는 노드가 정리되고, 미니맵과 줌 컨트롤로 넓은 스키마도 손쉽게 탐색할 수 있어요.

<Callout type="success">
편집할 때마다 약 1초 뒤 서버에 자동 저장돼요. 다른 기기에서 같은 계정으로 로그인해도 가장 최근 상태를 바로 이어서 볼 수 있습니다.
</Callout>

## 🤖 AI와 협업하는 방법

하단 중앙 채팅 입력창에 원하는 구조를 설명하면, AI가 **한국어 설명 + JSON 스키마**를 돌려줘요. JSON이 완전하면 다이어그램이 그 내용으로 교체됩니다.

![AI 채팅 응답 예시](/images/example.png)

- Free 플랜 기준으로 하루 50회까지 요청할 수 있어요. 사용량은 계정 단위로 합산되니 중요한 요청을 우선순위에 맞춰 작성하세요.
- 기존 구조를 보강하고 싶다면 메시지를 `현재 ERD 상태:`로 시작하고, 현재 스키마를 붙여 넣은 뒤 원하는 변경을 설명해 보세요.
- 결과를 부분적으로만 쓰고 싶다면 응답 JSON을 복사해 필요한 엔티티만 직접 편집하거나, 수정한 JSON을 다시 **스키마 가져오기**로 불러오면 돼요.

<Callout type="warning">
AI가 새 구조를 반영하면 이전 다이어그램은 덮어써요. 중요한 버전은 먼저 내보내기 탭에서 SQL이나 Prisma 파일로 백업해 두세요.
</Callout>

## 📤 산출물을 팀에 전달하기

`스키마 내보내기` 다이얼로그에서 MySQL, PostgreSQL, Prisma, JPA 중 원하는 형식을 고르면, 옵션에 맞춘 결과물을 복사하거나 파일로 받을 수 있어요. 외래키·인덱스·타임스탬프 포함 여부를 각 탭에서 조정할 수 있으니 팀 규칙에 따라 산출물을 맞춰보세요.

![스키마 내보내기 탭](/images/example.png)

<Callout type="info">
PNG 내보내기 버튼은 UI에 먼저 노출돼 있지만 아직 구현 전이에요. 이미지가 필요하면 브라우저 캡처 기능을 사용해주세요.
</Callout>
