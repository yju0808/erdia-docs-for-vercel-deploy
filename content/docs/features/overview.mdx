# 🎯 기능 한눈에 보기

Erdia는 개발자가 ERD를 설계할 때 거치는 과정을 기준으로 UI를 구성했어요. 아래 순서를 따라가면 대시보드에서 아이디어를 준비하고, 캔버스에서 구조를 다듬은 뒤, 팀에 공유하기까지 어떤 도구를 사용하면 되는지 빠르게 파악할 수 있습니다.

![주요 기능 흐름 미리보기](/images/example.png)

---

## 🗂 프로젝트를 정리할 때

대시보드는 모든 프로젝트를 최근 수정 순으로 보여줘요. 상단 검색창에서 이름이나 설명을 입력하면 목록이 실시간으로 필터링되고, `새 프로젝트` 버튼으로 즉시 빈 다이어그램을 만들 수 있습니다. 각 카드에는 설명과 마지막 저장 시간이 표시돼서, 팀원에게 현재 상태를 공유할 때도 유용해요.

<Callout type="info">
프로젝트를 생성하거나 수정하면 토스트로 결과가 즉시 표시돼요. 실패 메시지에 적힌 내용을 참고하면 입력 누락이나 중복 이름을 바로 해결할 수 있습니다.
</Callout>

---

## 🎨 캔버스에서 구조를 다듬을 때

편집기 상단 툴바는 엔티티 추가, 자동 정렬, 스키마 가져오기/내보내기 같은 핵심 기능을 한 자리에 모아두었어요. 엔티티 카드를 더블클릭하면 이름을 수정할 수 있고, 컬럼 옆 연필 아이콘으로 타입과 제약조건을 조정할 수 있습니다. 컬럼 핸들을 드래그하면 관계 설정 마법사가 열려 카디널리티와 식별 여부, 외래키 매핑을 순서대로 고를 수 있어요.

![캔버스 편집 흐름](/images/example.png)

<Callout type="success">
자동 정렬을 누르면 다이어그램 전체가 겹치지 않게 재배치되고, 미니맵과 줌 컨트롤로 넓은 스키마도 쉽게 탐색할 수 있어요. 변경 내용은 1초 이내에 자동 저장됩니다.
</Callout>

---

## 🤖 AI의 도움을 받을 때

하단 채팅 입력창에서 자연어로 요청하면 AI가 한국어 설명과 JSON 스키마를 함께 반환해요. JSON이 완전한 구조라면 다이어그램이 곧바로 그 상태로 교체되고, 우측 사이드바에는 대화 기록이 날짜별로 정리됩니다. Free 플랜에서는 하루 50회까지 요청할 수 있으니, 한도가 가까워지면 사이드바를 참고해 메시지를 조정해 주세요.

![AI 응답과 JSON 카드](/images/example.png)

<Callout type="warning">
AI가 제안한 결과를 그대로 적용하면 이전 다이어그램은 덮어써요. 중요한 버전은 내보내기로 백업한 뒤 실험해 보세요.
</Callout>

---

## 📤 결과를 공유할 때

`스키마 내보내기` 다이얼로그에서는 MySQL, PostgreSQL, Prisma, JPA 네 가지 형식을 지원해요. 스키마 이름과 외래키·인덱스·타임스탬프 포함 여부를 선택하고, 미리보기에서 생성된 코드를 확인한 뒤 복사하거나 파일로 저장할 수 있습니다. 반대로 기존 시스템을 들여오려면 `스키마 가져오기` 탭에서 동일한 형식을 붙여넣거나 파일로 업로드하면 돼요.

![가져오기와 내보내기 옵션](/images/example.png)

<Callout type="info">
PNG 내보내기 버튼은 아직 구현 전이에요. 이미지 공유가 필요하면 브라우저 캡처 기능을 이용해 주세요.
</Callout>
