# 🤖 AI 어시스턴트 사용법

Erdia는 OpenAI Chat Completions API를 활용해 자연어 설명을 ERD 구조로 변환합니다. 편집기 하단의 채팅 입력창을 통해 언제든지 어시스턴트에게 설계를 요청하거나 수정 지시를 내릴 수 있습니다.

## 🗣️ 대화 시작하기

1. 편집기 하단 중앙 입력창에 원하는 구조를 한국어나 영어로 설명합니다.
2. **Enter** 키 또는 우측 전송 버튼을 누르면 메시지가 전송되고 진행 상태가 표시됩니다.
3. 응답이 수신되면 우측에 채팅 사이드바가 자동으로 열려 이전 대화를 함께 확인할 수 있습니다.

<Callout type="info">
사이드바는 우측 하단 말풍선 버튼으로 언제든 열고 닫을 수 있습니다. 모바일에서는 드로어 형태로 전체 화면을 활용합니다.
</Callout>

## 🧠 AI가 제공하는 결과

- 어시스턴트는 항상 한국어 설명과 함께 JSON 스키마를 제공하도록 안내되어 있습니다.
- 응답 본문에 유효한 JSON이 포함되어 있으면 다이어그램이 자동으로 해당 구조로 재구성됩니다.
- JSON이 없거나 파싱에 실패하면 기존 다이어그램은 유지되고 텍스트만 채팅으로 남습니다.

<Callout type="warning">
자동 반영 기능은 현재 다이어그램을 통째로 덮어씁니다. 중요한 작업물일 경우 먼저 JSON으로 내보내 백업한 뒤 요청하세요.
</Callout>

## ✍️ 효과적인 프롬프트 작성 팁

- **새 다이어그램 생성**: "온라인 서점 ERD를 만들어줘. 회원, 주문, 결제 정보를 포함해줘"와 같이 원하는 엔티티와 관계를 설명합니다.
- **기존 다이어그램 수정**: 메시지에 `현재 ERD 상태:` 라벨을 포함하면 어시스턴트가 현재 구조를 읽고 변경 사항만 제안합니다.
- **세부 조건 명시**: "Order 테이블에 status 컬럼을 추가하고 ENUM 타입으로 설정해줘"처럼 컬럼 타입·제약 조건을 구체적으로 적어주세요.

## 🔐 환경 구성 주의사항

- 어시스턴트는 서버 사이드 라우트(`/api/chat`)에서 OpenAI API 키를 사용합니다. 키가 설정되지 않으면 500 오류가 발생합니다.
- 요청 실패 시 채팅 창에 `Failed to fetch response from server.` 메시지가 표시되며, 토큰 제한이나 네트워크 오류를 확인해야 합니다.
- 회사 정책상 외부로 공유하면 안 되는 데이터는 입력하지 말고, 필요한 경우 익명화된 예시 데이터를 사용하세요.

## 💡 대화 기록 활용

- 채팅 사이드바에는 날짜별로 메시지가 그룹화되어 재사용하기 쉽습니다.
- JSON 코드 블록은 별도 카드로 렌더링되어 복사하기 편합니다.
- 새로운 요청을 보낼 때 이전 응답을 참고하려면 사이드바에서 원하는 메시지를 스크롤해 확인하세요.
