# 📁 프로젝트 관리

Erdia 대시보드는 여러 ERD 설계를 한 곳에서 관리하도록 설계되었습니다. 프로젝트는 브라우저 로컬 스토리지에 저장되며, 최신 수정일 기준으로 정렬됩니다.

## 🔍 프로젝트 찾기

- 상단 검색창에 키워드를 입력하면 이름과 설명을 동시에 필터링합니다.
- 결과가 없을 때는 "검색 결과가 없습니다" 안내 카드가 표시되어, 새 프로젝트 버튼으로 바로 이동할 수 있습니다.

## 🆕 새 프로젝트 생성

1. 대시보드 우측 상단의 **`새 프로젝트`** 버튼을 선택합니다.
2. 프로젝트 이름(필수)과 설명(선택)을 입력합니다.
3. 동일한 이름이 존재하면 `중복된 이름` 토스트가 나타나고 저장이 차단됩니다.
4. 생성이 완료되면 성공 토스트가 노출되고, 새 프로젝트 상세 페이지로 자동 이동합니다.

<Callout type="info">
프로젝트는 `project-<타임스탬프>-<랜덤값>` 형식의 고유 ID를 사용합니다. 생성 즉시 빈 다이어그램이 포함되어 있으며, 필요하면 가져오기 기능으로 다른 스키마를 불러올 수 있습니다.
</Callout>

## ✏️ 프로젝트 정보 수정

- 프로젝트 카드의 점 3개 버튼 → **`수정`**을 선택하면 이름과 설명을 변경할 수 있는 다이얼로그가 열립니다.
- 이름은 필수이며 중복 검사를 통과해야 합니다.
- 저장 시 `프로젝트 수정 완료` 토스트가 노출되고 목록이 즉시 갱신됩니다.

## 🗑️ 프로젝트 삭제

1. 프로젝트 카드의 점 3개 버튼 → **`삭제`**를 선택합니다.
2. 확인 다이얼로그에서 삭제 시 영향을 받는 엔티티·관계 수를 안내합니다.
3. 삭제를 승인하면 로컬 스토리지에서 즉시 제거되고, `프로젝트 삭제 완료` 토스트가 표시됩니다.

<Callout type="warning">
삭제는 되돌릴 수 없습니다. 필요한 경우 삭제 전에 JSON 또는 SQL로 내보낸 뒤 안전하게 보관하세요.
</Callout>

## 🔄 자동 정렬 및 상태 표시

- 새로 생성하거나 수정된 프로젝트는 `updatedAt` 기준으로 자동 정렬됩니다.
- 각 카드에는 엔티티 수와 관계 수가 함께 표시되어, 프로젝트 규모를 빠르게 파악할 수 있습니다.

## 💾 저장 위치

모든 프로젝트 데이터는 브라우저의 `localStorage` 키 `erd-projects`에 보관됩니다. 다른 브라우저나 시크릿 모드에서는 확인할 수 없으므로, 중요한 다이어그램은 내보내기 후 별도 버전 관리 시스템에 보관하는 것을 권장합니다.
