# 🔄 가져오기 · 내보내기

Erdia는 표준 SQL 및 ORM 스키마를 활용해 다이어그램을 불러오거나 공유할 수 있도록 구성되어 있습니다. 모든 기능은 편집기 상단 툴바의 `더보기` 메뉴 또는 우측 상단 버튼을 통해 접근합니다.

## 📥 스키마 가져오기

1. `더보기` → **`스키마 가져오기`**를 선택하면 탭 형태의 다이얼로그가 열립니다.
2. 탭에서 대상 형식을 고릅니다: **MySQL · PostgreSQL · Prisma · JPA(Java)**.
3. 스키마 텍스트를 붙여넣거나, `.sql`, `.prisma`, `.java` 파일을 업로드합니다.
4. 옵션을 조정합니다.
   - **엔티티 자동 배치**: 가져온 엔티티 위치를 자동으로 배치합니다.
   - **관계 가져오기**: 관계 정보를 함께 생성할지 여부를 설정합니다.
5. **`가져오기`** 버튼을 누르면 엔티티와 관계 수를 요약한 성공 토스트가 나타납니다. 실패 시 형식 오류를 알리는 경고 토스트가 출력됩니다.

<Callout type="warning">
현재는 스키마 파싱 오류가 발생하면 다이어그램이 변경되지 않습니다. 오류 메시지를 확인한 뒤 SQL 문법이나 Prisma 모델 정의를 다시 검토해주세요.
</Callout>

## 📤 스키마 내보내기

1. 상단 툴바의 **`스키마 내보내기`** 버튼을 선택합니다.
2. 동일한 네 가지 형식(MySQL, PostgreSQL, Prisma, JPA) 중 원하는 탭을 고릅니다.
3. 옵션을 설정합니다.
   - **스키마 이름**: 데이터베이스 또는 패키지 이름을 입력합니다.
   - **타임스탬프 포함**: created_at, updated_at 필드를 추가합니다.
   - **외래키 포함**: FK 제약 조건을 포함합니다.
   - **인덱스 포함**: SQL 탭에서 인덱스 문을 생성합니다.
4. 미리보기 패널에서 생성된 스키마를 확인한 뒤, `복사` 또는 `다운로드` 버튼으로 결과를 저장합니다.

<Callout type="info">
내보낸 파일은 텍스트 기반이므로 Git 등의 형상 관리 도구에 바로 저장할 수 있습니다. 다운로드 후 코드 리뷰 워크플로에 첨부해 보세요.
</Callout>

## 🖼️ PNG 내보내기 (준비 중)

툴바의 `PNG 내보내기` 버튼은 향후 업데이트를 위한 자리표시자입니다. 현재 버전에서는 클릭해도 파일이 생성되지 않으니, 이미지가 필요하면 브라우저 캡처 기능을 사용해 주세요.

## 💾 백업 모범 사례

- 정기적으로 SQL 또는 JSON으로 내보내 별도 버전 관리 저장소에 보관하세요.
- 팀과 공유할 때는 형식별 옵션을 명확히 지정해 불필요한 컬럼이나 제약조건이 포함되지 않도록 합니다.
- 대규모 스키마를 가져온 뒤에는 자동 배치 후 수동으로 노드 위치를 조정해 가독성을 확보하세요.
