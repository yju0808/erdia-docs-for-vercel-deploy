# 🎨 캔버스 & 편집기

Erdia 편집기는 React Flow 기반 캔버스 위에서 엔티티와 관계를 조작할 수 있도록 구성되었습니다. 아래 내용을 따라 기본 편집 흐름과 생산성 도구를 익히세요.

## 🧱 화면 구성

- **상단 툴바** – 프로젝트 이름·설명을 보여주고 `엔티티 추가`, `자동 정렬`, `더보기` 메뉴(내보내기·가져오기·전체 삭제)를 제공합니다.
- **우측 하단 플로팅 버튼** – 채팅 사이드바 열기, 테마 토글, 빠른 엔티티 추가 버튼이 배치되어 있습니다.
- **하단 중앙 채팅 입력창** – AI 어시스턴트에 자연어 요청을 보낼 수 있는 공간입니다.
- **React Flow 캔버스** – 엔티티 카드와 관계선을 배치하며, 미니맵·줌 컨트롤이 제공됩니다.

## ➕ 엔티티 다루기

1. 상단 툴바의 **`엔티티 추가`** 버튼을 누르거나 캔버스 빈 공간을 우클릭해 **`여기에 엔티티 추가`**를 선택합니다.
2. 새 엔티티는 `New Entity` 이름과 `id` 기본키 속성을 포함한 상태로 생성됩니다.
3. 제목을 더블클릭하면 이름을 편집할 수 있으며, `완료` 버튼을 누르거나 포커스를 잃으면 저장됩니다.
4. 각 속성 행의 연필 아이콘을 클릭하면 속성 편집 패널이 열립니다.
   - **타입 변경**: INT, VARCHAR, TEXT, DATE, BOOLEAN, DECIMAL 지원
   - **기본값**: 타입별로 허용 형식 검증
   - **기본키/외래키/필수값 토글**: 스위치로 즉시 변경
   - **삭제**: 휴지통 아이콘을 클릭하면 삭제 확인 다이얼로그가 표시됩니다.
5. 엔티티 카드 우측 상단 휴지통 아이콘을 누르면 전체 엔티티 삭제를 요청하고, 연결된 관계 수에 따라 영향 범위를 안내합니다.

<Callout type="info">
엔티티를 우클릭하면 **컨텍스트 메뉴**가 열려 "엔티티 편집", "엔티티 삭제"를 빠르게 실행할 수 있습니다.
</Callout>

## 🔗 관계 만들기

1. 엔티티 속성 옆 원형 핸들을 드래그해 다른 엔티티로 연결을 시작합니다.
2. 드래그가 시작되면 상단에 연결 가이드 배지가 나타나 현재가 "출발"인지 "도착"인지 알려줍니다.
3. 연결을 놓으면 **관계 타입 선택** 다이얼로그가 열립니다.
   - 일대일(1:1), 일대다(1:N), 영 또는 일(0..1), 영 또는 다수(0..N)
   - 식별 관계 여부 토글 (식별: 실선, 비식별: 점선)
4. 타입을 선택하면 **속성 매핑 다이얼로그**가 열립니다.
   - 소스/타깃 엔티티의 속성을 선택하거나 "없음"을 선택해 엔티티 레벨 관계로 유지할 수 있습니다.
   - 일대다 관계에서는 대상 엔티티의 외래키 후보를 자동으로 추천합니다.
5. 완료 후 관계선 위의 설정 버튼을 통해 언제든지 관계 타입과 표기(1, N, 0..1 등)를 다시 조정할 수 있습니다.

## 🗑️ 삭제 & 초기화 옵션

- Delete 또는 Backspace 키로 선택한 엔티티·관계를 삭제할 수 있으며, 항상 확인 다이얼로그가 표시됩니다.
- 상단 툴바 `더보기` → `전체 삭제`를 선택하면 다이어그램을 비우기 전에 영향 범위를 알려줍니다.
- 속성 삭제 역시 확인 단계를 거쳐 실수로 인한 데이터 손실을 방지합니다.

## 🔄 자동 정렬과 보기 제어

- 상단 툴바의 **`자동 정렬`**을 누르면 Dagre 기반 레이아웃을 적용한 뒤 겹침을 자동으로 해소합니다.
- 정렬이 끝나면 캔버스 뷰가 전체 다이어그램을 보기 좋게 맞춥니다.
- 미니맵과 줌 컨트롤로 넓은 다이어그램을 빠르게 탐색할 수 있습니다.

## 🎛️ 테마 & 저장 상태

- 우측 하단 태양/달 아이콘을 눌러 라이트/다크 테마를 전환하면 선택한 테마가 로컬 스토리지에 저장됩니다.
- 상단 툴바 우측의 저장 표시가 `저장 중...` → `저장 완료`로 변경되어 자동 저장 상태를 실시간 확인할 수 있습니다.

## ⌨️ 단축키 요약

- **Delete / Backspace** – 선택 항목 삭제 (확인 다이얼로그 표시)
- **Esc** – 열려 있는 컨텍스트 메뉴, 다이얼로그, 편집 모드를 닫습니다.
- **Enter** – 엔티티 이름 입력 후 저장

<Callout type="success">
단축키는 입력 필드나 다이얼로그가 열려 있지 않을 때만 동작합니다. 편집 중 실수로 삭제되지 않도록 보호 로직이 적용되어 있습니다.
</Callout>
