# 💬 자주 묻는 질문

Erdia를 사용하면서 자주 궁금해하시는 내용을 정리했어요. 아래 질문에서 원하는 답을 찾아보세요.

---

## 🙋 로그인은 꼭 해야 하나요?

네, 현재 버전은 Google 로그인으로 이용해요. 홈 화면에서 `무료로 시작하기`를 누르면 로그인 페이지가 열리고, `Google로 계속하기` 버튼으로 인증을 마치면 대시보드로 이동합니다.

![Google 로그인 안내](/images/example.png)

<Callout type="info">
로그인 화면의 "게스트 모드로 둘러보기" 버튼은 아직 준비 중이에요. 정식으로 체험하려면 Google 로그인을 이용해주세요.
</Callout>

---

## 🗄️ 데이터는 어디에 저장되나요?

모든 프로젝트와 다이어그램은 서버에 자동 저장돼요. 편집기 상단의 `저장 중...` 배지가 `저장 완료`로 바뀌면 서버 반영이 끝났다는 의미입니다. 

같은 계정으로 로그인하면 다른 기기에서도 바로 이어서 작업할 수 있어요.

<Callout type="success">
변경 후 약 1초 이내에 자동 저장이 실행됩니다. 네트워크가 안정적이라면 별도로 저장 버튼을 누를 필요가 없어요.
</Callout>

---

## 🤔 AI가 제안한 다이어그램을 일부만 쓰고 싶어요

AI 응답은 JSON 스키마 전체를 보내기 때문에 자동 반영 시 다이어그램이 통째로 교체돼요. 일부만 적용하려면 다음 방법을 사용하세요:

**방법 1: 수동 편집**
- 다이어그램 전체를 다시 그려달라고 하기보다는, "User 테이블에 email 컬럼이 필요해" 같은 작은 단위로 요청해요
- AI 응답을 참고해 필요한 테이블과 컬럼만 인터랙티브 캔버스에서 직접 수정해요

**방법 2: JSON 편집 후 가져오기**
- AI가 준 JSON을 복사해서 필요한 부분만 남기고 편집한 뒤
- 상단 툴바의 `스키마 가져오기`로 다시 불러와요

<Callout type="warning">
AI가 새 구조를 반영하면 이전 다이어그램은 덮어써요. 중요한 버전은 먼저 내보내기로 백업해 두세요.
</Callout>

## 🔗 엔티티나 컬럼을 삭제하면 관계도 사라지나요?

네, **엔티티를 삭제하면 연결된 모든 관계도 함께 지워집니다**.

컬럼을 삭제할 때는 기본키나 외래키 여부에 따라 경고 메시지가 먼저 나타나요:

| 컬럼 타입 | 경고 메시지 |
|---------|----------|
| 기본키 (Primary Key) | "경고: 기본키입니다. 삭제 시 관계에 영향을 줄 수 있습니다." |
| 외래키 (Foreign Key) | "경고: 외래키입니다. 삭제 시 관계에 영향을 줄 수 있습니다." |
| 일반 컬럼 | 바로 삭제 |

<Callout type="warn">
실수로 삭제한 경우 즉시 되돌릴 수 없어요. 삭제 전 다이어그램을 내보내기로 백업하는 습관을 추천해요.
</Callout>

## 🤝 팀원과 어떻게 공유하나요?

상단 툴바의 `스키마 내보내기`에서 MySQL, PostgreSQL, Prisma, JPA 중 하나를 골라 복사하거나 다운로드하세요.

![스키마 내보내기 버튼 위치](/images/example.png)

**내보내기 옵션:**
- **스키마 이름**: 데이터베이스 이름 지정
- **타임스탬프 포함**: created_at, updated_at 자동 추가
- **외래키 포함**: FOREIGN KEY 제약조건 생성
- **인덱스 포함**: INDEX 생성 (MySQL, PostgreSQL만 해당)

상대 팀은 동일한 형식을 `스키마 가져오기`로 불러오면 같은 구조를 확인할 수 있습니다.

<Callout type="info">
현재는 파일로 공유하는 방식만 지원해요. Pro 플랜에서는 링크로 공유하고 실시간 협업할 수 있도록 준비 중이에요.
</Callout>

## 🖼️ PNG 내보내기는 언제 되나요?

툴바에 버튼이 있지만 아직 구현 전이에요. 업데이트가 배포되면 문서와 릴리즈 노트에서 함께 안내할 예정입니다.

<Callout type="warn">
PNG 내보내기 버튼을 클릭해도 아무 동작도 하지 않아요. 이미지 공유가 필요하면 브라우저 캡처 기능을 사용해 주세요.
</Callout>

**임시 해결책:**
- 브라우저의 전체 화면 캡처 (F11)
- 스크린샷 도구 (Windows: Win+Shift+S, macOS: Cmd+Shift+4)
- 브라우저 확장 프로그램 (Awesome Screenshot 등)
