# 💬 자주 묻는 질문

## 🙋 회원가입이나 로그인이 필요한가요?

아니요. 현재 버전은 계정 시스템 없이 작동하며, 브라우저 로컬 스토리지에 프로젝트가 저장됩니다. 홈 화면에서 `시작하기` 버튼만 누르면 대시보드로 이동해 바로 작업을 시작할 수 있습니다.

## 🗄️ 데이터는 어디에 저장되나요?

모든 프로젝트는 브라우저 `localStorage`의 `erd-projects` 키 아래에 저장됩니다. 같은 브라우저에서만 접근 가능하므로, 다른 기기에서 작업하려면 SQL/JSON으로 내보낸 뒤 가져오기 기능을 사용하세요.

<Callout type="warning">
브라우저 캐시를 지우거나 시크릿 모드를 사용하면 데이터가 사라질 수 있습니다. 중요한 다이어그램은 정기적으로 백업하세요.
</Callout>

## 🤔 AI가 제안한 다이어그램을 일부만 적용할 수 있나요?

AI 어시스턴트가 반환한 JSON 스키마는 현재 다이어그램 전체를 대체합니다. 일부만 반영하고 싶다면 응답을 참고해 수동으로 엔티티·속성을 추가하거나, JSON을 편집한 뒤 가져오기 기능을 활용하세요.

## 🔗 관계나 속성을 삭제하면 연결이 모두 사라지나요?

삭제 전 확인 다이얼로그에서 영향 범위를 안내합니다. 엔티티를 삭제하면 연결된 관계도 함께 제거되며, 속성 삭제 시 기본키 또는 외래키인지 여부에 따라 경고 메시지가 추가로 표시됩니다.

## 🤝 팀과 다이어그램을 공유하려면 어떻게 하나요?

상단 툴바의 `스키마 내보내기`를 사용해 MySQL, PostgreSQL, Prisma, JPA 형식으로 변환한 뒤 파일을 공유하세요. 상대가 동일한 형식을 사용한다면 `스키마 가져오기`로 동일한 다이어그램을 복원할 수 있습니다.

## 🖼️ PNG 내보내기는 지원하나요?

툴바에 `PNG 내보내기` 버튼이 있지만, 현재는 자리표시자 상태입니다. 이미지를 공유하려면 브라우저 캡처 기능을 사용하거나, 내보낸 SQL을 기반으로 외부 도구에서 다이어그램을 생성해 주세요.
