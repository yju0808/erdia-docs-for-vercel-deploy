# 자주 묻는 질문 (FAQ)

Erdia 사용 중 궁금한 점을 빠르게 찾아보세요.

## 💬 일반 질문

### Erdia는 무료인가요?

네! 현재 모든 기능을 **완전 무료**로 제공하고 있습니다. 향후 실시간 협업 기능이 추가되는 Pro 플랜이 출시될 예정이지만, Free 플랜은 계속 유지됩니다.

<Callout type="success">
Free 플랜은 **영구적으로 무료**이며, 신용카드 등록 없이 사용할 수 있습니다.
</Callout>

---

### 회원가입 없이 사용할 수 있나요?

회원가입은 필수입니다. 하지만 Google OAuth를 사용하여 **클릭 한 번**으로 간편하게 가입할 수 있으며, 클라우드 저장을 위해 계정이 필요합니다.

---

### 몇 개의 프로젝트를 만들 수 있나요?

Free 플랜에서도 **무제한 프로젝트**를 생성할 수 있습니다.

---

### AI 채팅은 몇 번까지 사용할 수 있나요?

Free 플랜에서는 **하루 50회**까지 사용할 수 있습니다. Pro 플랜(추후 출시)에서는 500회까지 가능합니다.

<Callout type="info">
사용 횟수는 매일 자정(00:00 KST)에 초기화됩니다.
</Callout>

---

### 데이터는 안전하게 보관되나요?

네! 모든 데이터는 클라우드 서버에 **암호화**되어 저장됩니다. OAuth2 + JWT 기반 인증으로 보안을 강화했습니다.

**보안 조치:**
- 🔒 데이터 전송 시 HTTPS 암호화
- 🔐 JWT 토큰 기반 인증
- ☁️ AWS 클라우드 서버 사용
- 💾 실시간 자동 백업

---

## 🛠 기능 관련 질문

### 어떤 데이터베이스를 지원하나요?

현재 **MySQL**과 **PostgreSQL**을 지원합니다.

| 데이터베이스 | 지원 여부 | 상태 |
|-------------|----------|------|
| MySQL | ✅ | 지원 |
| PostgreSQL | ✅ | 지원 |
| MariaDB | 🔜 | 준비 중 |
| SQLite | 🔜 | 준비 중 |
| MSSQL | 🔜 | 계획 중 |
| Oracle | 🔜 | 계획 중 |

---

### AI가 생성한 ERD를 수정할 수 있나요?

네! AI가 생성한 ERD는 캔버스에서 **자유롭게 수정**할 수 있습니다.

**가능한 편집:**
- ✅ 엔티티 추가/삭제
- ✅ 속성 변경
- ✅ 관계 수정
- ✅ 데이터 타입 변경
- ✅ 제약조건 설정

---

### 기존 ERD를 가져올 수 있나요?

네! **JSON 파일**로 백업한 ERD를 가져올 수 있습니다. 현재는 Erdia 자체 JSON 포맷만 지원하며, 향후 다른 ERD 도구의 파일 형식도 지원할 예정입니다.

<Callout type="info">
**지원 예정 형식**: MySQL Workbench (.mwb), dbdiagram.io, ERDCloud 등
</Callout>

---

### 생성된 SQL을 바로 실행할 수 있나요?

SQL 파일을 다운로드한 후 MySQL 또는 PostgreSQL 클라이언트에서 실행하셔야 합니다.

**실행 방법:**

**MySQL:**
```bash
mysql -u username -p database_name < schema.sql
```

**PostgreSQL:**
```bash
psql -U username -d database_name -f schema.sql
```

<Callout type="info">
**향후 업데이트**: DB 직접 연결 기능이 추가될 예정입니다.
</Callout>

---

### 팀원과 협업할 수 있나요?

현재는 개인 작업만 가능합니다. **실시간 협업 기능**은 Pro 플랜 출시와 함께 제공될 예정입니다 (2025년 하반기).

**예정된 협업 기능:**
- 👥 실시간 멀티플레이어 편집
- 💬 댓글 및 피드백
- 📜 버전 관리
- 🔔 변경 알림

## 🔧 기술 관련 질문

### 어떤 브라우저를 지원하나요?

최신 버전의 Chrome, Firefox, Safari, Edge를 지원합니다.

| 브라우저 | 지원 여부 | 권장도 |
|---------|----------|--------|
| Chrome | ✅ | ⭐⭐⭐ 최고 |
| Firefox | ✅ | ⭐⭐⭐ 우수 |
| Safari | ✅ | ⭐⭐ 양호 |
| Edge | ✅ | ⭐⭐⭐ 우수 |

<Callout type="success">
최상의 경험을 위해 **Chrome**을 권장합니다.
</Callout>

---

### 모바일에서도 사용할 수 있나요?

네! 반응형 디자인으로 모바일 브라우저에서도 사용 가능합니다. 다만 복잡한 편집 작업은 **PC 환경**을 권장합니다.

**모바일 사용 시 제약:**
- 작은 화면으로 인한 가독성 저하
- 터치 조작의 정밀도 부족
- 복잡한 ERD 편집의 어려움

---

### 오프라인에서도 작업할 수 있나요?

아니요. Erdia는 클라우드 기반 서비스로 **인터넷 연결**이 필요합니다. 오프라인 모드는 향후 고려 중입니다.

---

### 데이터는 어디에 저장되나요?

모든 프로젝트 데이터는 **AWS 클라우드 서버**에 저장됩니다. 실시간 자동 저장으로 데이터 손실 걱정이 없습니다.

**저장 위치:**
- 🌐 AWS 서울 리전 (ap-northeast-2)
- 💾 PostgreSQL 데이터베이스
- 🔄 실시간 자동 백업

---

### API를 제공하나요?

현재는 제공하지 않습니다. 추후 Enterprise 플랜에서 **REST API**를 제공할 예정입니다.

**예정된 API 기능:**
- ERD 생성/조회/수정/삭제
- SQL 내보내기
- JSON 가져오기/내보내기

---

## 🚨 문제 해결

### ERD가 저장되지 않아요

다음을 확인해주세요:

1. ✅ 인터넷 연결 상태 확인
2. ✅ 브라우저 쿠키 활성화 여부 확인
3. ✅ 툴바 상단의 저장 상태 확인 ("저장 중..." 또는 "저장됨")
4. ✅ 문제가 계속되면 hello@erdia.app로 문의

<Callout type="warning">
브라우저의 시크릿 모드에서는 쿠키 설정에 따라 저장이 안 될 수 있습니다.
</Callout>

---

### AI 채팅이 응답하지 않아요

다음을 시도해보세요:

1. 🔄 페이지 새로고침 (F5)
2. 📊 하루 사용 횟수 확인 (Free: 50회/일)
3. 📝 요청이 너무 복잡하지 않은지 확인 (한 번에 3~5개 테이블 권장)
4. ⏰ 잠시 후 다시 시도

**여전히 문제가 있다면:**
- 브라우저 콘솔(F12)에서 에러 확인
- 다른 브라우저로 시도
- hello@erdia.app로 문의

---

### 로그인이 안 돼요

다음을 확인해주세요:

1. ✅ Google 계정 로그인 상태 확인
2. ✅ 브라우저 팝업 차단 해제
3. ✅ 시크릿 모드 사용 시 쿠키 허용 확인
4. ✅ 다른 브라우저로 시도

**팝업 차단 해제 방법:**
- Chrome: 주소창 우측 팝업 차단 아이콘 클릭 → "항상 허용"
- Firefox: 주소창 좌측 설정 아이콘 → "팝업 및 리디렉션 허용"

---

### 내보낸 SQL이 실행되지 않아요

다음을 확인해주세요:

1. ✅ 올바른 데이터베이스 엔진 선택 (MySQL vs PostgreSQL)
2. ✅ 스키마 이름이 실제 DB와 일치하는지 확인
3. ✅ Foreign Key 제약조건 순서 확인 (부모 테이블이 먼저 생성되어야 함)
4. ✅ 데이터베이스 권한 확인

**SQL 실행 오류 예시:**

**오류:** `Table 'mydb.Post' doesn't exist`
**해결:** 부모 테이블(User)이 먼저 생성되었는지 확인

**오류:** `Access denied for user`
**해결:** 데이터베이스 계정에 CREATE, ALTER 권한이 있는지 확인

---

### 프로젝트를 삭제했는데 복구할 수 있나요?

아니요. 삭제된 프로젝트는 **복구할 수 없습니다**. 중요한 프로젝트는 삭제 전 **JSON 백업**을 권장합니다.

<Callout type="danger">
**주의**: 프로젝트 삭제는 되돌릴 수 없습니다. 삭제 전 반드시 백업하세요!
</Callout>

---

### 관계선이 연결되지 않아요

다음을 확인해주세요:

1. ✅ 두 속성의 데이터 타입이 호환되는지 확인
2. ✅ Foreign Key는 Primary Key와 같은 타입이어야 함
3. ✅ 드래그 동작이 정확한지 확인

**호환되는 타입 예시:**
- INT ↔ INT ✅
- BIGINT ↔ BIGINT ✅
- VARCHAR(100) ↔ VARCHAR(255) ✅
- INT ↔ VARCHAR ❌
