# 캔버스 편집

Erdia의 직관적인 캔버스에서 ERD를 자유롭게 편집해보세요.

## 엔티티 노드 조작

### 엔티티 추가하기

새로운 테이블을 추가하는 방법은 두 가지입니다.

**방법 1: 버튼 사용**
1. 우측 하단의 **+ 버튼** 클릭
2. 엔티티 이름 입력
3. 속성 추가

**방법 2: 우클릭 메뉴**
1. 캔버스 빈 공간에서 **우클릭**
2. **"엔티티 추가"** 선택
3. 엔티티 정보 입력

<Callout type="info">
새로 추가된 엔티티는 캔버스 중앙에 생성됩니다. 원하는 위치로 드래그하여 이동할 수 있습니다.
</Callout>

### 엔티티 편집하기

엔티티를 클릭하면 편집 모드가 활성화됩니다.

**편집 가능한 항목:**

| 항목 | 설명 | 예시 |
|------|------|------|
| **테이블 이름** | 엔티티의 이름 | User, Product, Order |
| **속성(컬럼)** | 테이블의 필드 | id, email, name |
| **데이터 타입** | 각 속성의 타입 | INT, VARCHAR(255), TEXT |
| **제약조건** | PK, FK, Nullable, Unique | Primary Key 체크 |

**속성 추가하기:**
1. 엔티티 노드를 **클릭**하여 편집 모드 활성화
2. 속성 목록 하단의 **"+ 속성 추가"** 클릭
3. 컬럼 이름과 데이터 타입 입력
4. 제약조건 설정 (선택사항)

**속성 삭제하기:**
1. 속성 우측의 **🗑️ 아이콘** 클릭
2. 확인 다이얼로그에서 **"삭제"** 선택

<Callout type="warning">
Primary Key나 Foreign Key로 사용 중인 속성을 삭제하면 관련된 관계도 함께 삭제됩니다.
</Callout>

### 엔티티 삭제하기

더 이상 필요 없는 엔티티를 제거할 수 있습니다.

**방법 1: 우클릭 메뉴**
1. 엔티티 노드에서 **우클릭**
2. **"삭제"** 선택
3. 확인 다이얼로그에서 **"삭제 확인"**

**방법 2: 키보드 단축키**
1. 엔티티 노드를 **클릭**하여 선택
2. **Delete 키** 누르기
3. 확인 다이얼로그에서 **"삭제 확인"**

<Callout type="danger">
엔티티를 삭제하면 해당 엔티티와 연결된 모든 관계도 함께 삭제됩니다. 이 작업은 되돌릴 수 없으므로 신중하게 진행하세요.
</Callout>

## 관계 연결하기

엔티티 간의 관계를 시각적으로 표현할 수 있습니다.

### 1:N 관계 (One-to-Many)

한 엔티티가 다른 엔티티 여러 개와 연결되는 관계입니다.

**예시: User가 여러 Post를 작성**

1. User 테이블의 **`id` 속성**을 클릭한 채로 드래그
2. Post 테이블의 **`user_id` 속성**으로 드래그하여 연결
3. 관계 타입 선택 다이얼로그에서 **"1:N"** 선택
4. 식별 관계 여부 선택:
   - **식별 관계**: `user_id`가 Post의 Primary Key에 포함됨
   - **비식별 관계**: `user_id`가 일반 Foreign Key

<Callout type="success">
**식별 관계 vs 비식별 관계**

- **식별 관계**: 부모 테이블의 PK가 자식 테이블의 PK에 포함 (강한 종속성)
- **비식별 관계**: 부모 테이블의 PK가 자식 테이블의 일반 FK (약한 종속성)

대부분의 경우 **비식별 관계**를 사용합니다.
</Callout>

### N:M 관계 (Many-to-Many)

여러 엔티티가 여러 엔티티와 연결되는 관계입니다.

**예시: Post와 Tag의 다대다 관계**

N:M 관계는 중간 테이블을 통해 구현합니다.

**방법 1: AI에게 요청**
```
💬 "Post와 Tag의 다대다 관계 만들어줘"
```
AI가 자동으로 `PostTag` 중간 테이블을 생성합니다.

**방법 2: 수동 생성**
1. 중간 테이블 생성 (예: `PostTag`)
2. `post_id`, `tag_id` 속성 추가
3. `PostTag` ↔ `Post` 1:N 관계 연결
4. `PostTag` ↔ `Tag` 1:N 관계 연결

### 1:1 관계 (One-to-One)

한 엔티티가 다른 엔티티 하나와만 연결되는 관계입니다.

**예시: User와 Profile의 1:1 관계**

1. User 테이블의 **`id` 속성** 드래그
2. Profile 테이블의 **`user_id` 속성**에 연결
3. 관계 타입에서 **"1:1"** 선택
4. Profile의 `user_id`에 **UNIQUE 제약조건** 추가

### 관계 삭제하기

**방법 1: 직접 선택**
1. 관계 선(Edge)을 **클릭하여 선택**
2. **Delete 키** 누르기

**방법 2: 아이콘 클릭**
1. 관계 선 위에 마우스 올리기
2. 우측의 **🗑️ 아이콘** 클릭

## 자동 정렬 기능

복잡한 다이어그램을 깔끔하게 정리할 수 있습니다.

### 언제 사용하나요?

- ✅ 엔티티가 많아서 화면이 복잡할 때
- ✅ 관계선이 겹쳐서 가독성이 떨어질 때
- ✅ 새로 추가한 엔티티를 적절한 위치에 배치하고 싶을 때

### 사용 방법

1. 툴바에서 **"자동 정렬"** 버튼 클릭
2. Dagre 레이아웃 알고리즘이 엔티티를 자동으로 배치
3. 겹치는 노드를 감지하여 간격 조정

<Callout type="info">
**자동 정렬 알고리즘**

Erdia는 Dagre 레이아웃 알고리즘을 사용하여 다음 원칙으로 배치합니다:
- 계층 구조: 부모 엔티티가 위, 자식 엔티티가 아래
- 관계선 최소화: 선이 교차하지 않도록 배치
- 균등 간격: 엔티티 간 일정한 간격 유지
</Callout>

### 수동 조정

자동 정렬 후에도 엔티티를 드래그하여 원하는 위치로 이동할 수 있습니다.

## 노드 스타일링

### 색상 변경 (준비 중)

현재는 모든 엔티티가 동일한 색상으로 표시되지만, 향후 업데이트에서 테이블별 색상 지정 기능이 추가될 예정입니다.

### 아이콘 추가 (준비 중)

엔티티 타입에 따라 아이콘을 추가하는 기능이 준비 중입니다.

## 캔버스 네비게이션

### 확대/축소

**마우스 휠:**
- 휠을 위로: 확대
- 휠을 아래로: 축소

**툴바 버튼:**
- `+` 버튼: 확대
- `-` 버튼: 축소
- `⌘` 버튼: 전체 화면에 맞춤

### 이동

**드래그:**
캔버스 빈 공간을 드래그하여 화면을 이동할 수 있습니다.

**미니맵 (준비 중):**
향후 업데이트에서 미니맵 기능이 추가될 예정입니다.

## 실전 예시

### 쇼핑몰 ERD 편집하기

**1단계: AI로 기본 구조 생성**
```
💬 "쇼핑몰 ERD 만들어줘. User, Product, Order, OrderItem 테이블 필요해"
```

**2단계: User 테이블 편집**
1. User 엔티티 클릭
2. 속성 추가:
   - `phone` VARCHAR(20)
   - `address` TEXT
   - `postal_code` VARCHAR(10)

**3단계: 관계 연결**
1. User.id → Order.user_id (1:N)
2. Order.id → OrderItem.order_id (1:N)
3. Product.id → OrderItem.product_id (1:N)

**4단계: 자동 정렬**
툴바의 "자동 정렬" 버튼 클릭

**5단계: 미세 조정**
엔티티를 드래그하여 원하는 위치로 이동

## 유용한 팁 💡

### 1. 속성 순서 조정

속성을 드래그하여 순서를 변경할 수 있습니다.
- Primary Key를 항상 최상단에 배치
- Foreign Key를 그 다음에 배치
- 일반 속성은 그 아래에 배치

### 2. 명명 규칙 통일

일관된 명명 규칙을 사용하세요.
- 테이블명: PascalCase (User, OrderItem)
- 컬럼명: snake_case (user_id, created_at)

### 3. 주석 활용 (준비 중)

향후 업데이트에서 엔티티와 속성에 주석을 추가하는 기능이 제공될 예정입니다.

### 4. 그룹화 (준비 중)

관련된 엔티티를 그룹으로 묶는 기능이 준비 중입니다.

## 문제 해결

### 관계가 연결되지 않아요

- 두 속성의 데이터 타입이 호환되는지 확인하세요
- Foreign Key는 Primary Key와 같은 타입이어야 합니다

### 엔티티가 너무 많아서 복잡해요

1. 자동 정렬 기능 사용
2. 캔버스 확대/축소로 전체 구조 파악
3. 관련 없는 엔티티는 멀리 배치

### 실수로 삭제했어요

- 현재는 실행 취소 기능이 없습니다
- AI 채팅으로 다시 생성하거나 캔버스에서 직접 추가하세요
- 중요한 작업 전에는 JSON으로 백업하세요

<Callout type="info">
**향후 업데이트 예정**: 실행 취소/다시 실행 (Undo/Redo) 기능이 추가될 예정입니다.
</Callout>
